Customer Segmentation & Churn Prediction

Segmented 9,164 e-commerce customers using K-means (feature selection + PCA) and built an XGBoost churn model (92% accuracy) to support targeted marketing and retention strategies.

  • Tools: Python • SQL • Tableau
  • Methods: RFM • PCA • K-means • t-SNE • XGBoost
  • Deliverables: Segments • Insights • Model
← Back to Portfolio GitHub Repo

Problem

Marketing teams often struggle to decide who to prioritize and how to personalize messaging. This project groups customers into behavior-based segments and builds a churn prediction model to identify high-risk customers for proactive retention.

Data

  • Size: 9,164 customers
  • Time range: 2014–2021
  • Data type: Transaction history + product/category information
What I built: a customer-level table with behavior and value features (RFM + category spend) for clustering, plus a churn label for supervised modeling.

Feature Engineering

I aggregated transaction data to the customer level and created features that reflect customer value and behavior. The core framework is RFM:

  • Recency: how recently a customer purchased
  • Frequency: how often they purchased
  • Monetary: how much they spent

Additional features

  • Customer tenure
  • Average spend, total quantity, purchase cycle
  • Category spend (e.g., beverage, food, coffee, dishware, kitchen supplies, medicine)

Modeling Approach

1) Reduce redundancy (feature selection)

Because K-means relies on distance, highly correlated features can distort clustering. I removed redundant features using correlation checks to improve stability and interpretability.

2) Dimensionality reduction (PCA)

I also ran PCA to compress features into a smaller set of components while keeping most of the variance. This provides an alternative representation for clustering and helps visualize segment structure.

3) Clustering (K-means)

I used the elbow method to choose a practical number of clusters for business use (balanced separation + interpretability).

Segment Insights

The segments surfaced clear, actionable groups that map well to marketing strategies. Common patterns included:

High-value customers

  • High frequency + high spend + recent activity
  • Marketing action: loyalty/VIP offers, retention messaging, higher budget priority

At-risk customers

  • Purchases slowing down (recency increasing) while spend may still be meaningful
  • Marketing action: win-back campaigns, reminders, personalized incentives

Low-value / inactive customers

  • Low frequency + low spend + long time since last purchase
  • Marketing action: low-cost reactivation, exclude from expensive targeting

Category-driven segments

  • Clusters that over-index on certain product categories
  • Marketing action: tailor creative + offers around category interest

Churn Prediction

To complement segmentation, I trained an XGBoost model to predict churn risk and help prioritize retention outreach.

  • Model: XGBoost classifier
  • Outcome: 92% accuracy (as reported in the case summary)
  • Use: flag high-risk customers for targeted retention campaigns
How marketing can use this: combine segment + churn risk to decide (1) who to target, (2) what message/offer to use, and (3) how much budget to allocate.

Conclusion

This analysis produced customer segments that are easy to act on and a churn model to prioritize retention. Together, they support more efficient budget allocation, better personalization, and clearer lifecycle marketing strategy.

Note: Unsupervised clustering doesn’t have a single “accuracy” metric; I selected the final segmentation based on a combination of separation quality and business usefulness.
